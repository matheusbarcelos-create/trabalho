Requisitos


O projeto consiste no desenvolvimento de um site completo com no mínimo 10 páginas, abordando um tema livre à escolha do aluno. O objetivo é aplicar conceitos fundamentais de estruturação web, organização de arquivos e boas práticas de desenvolvimento.

O site deve possuir uma organização básica de arquivos, incluindo:
-Pasta principal do projeto (site)
-Subpasta para imagens
-Subpasta para arquivos de estilo (CSS)
-Arquivo principal index.html
-Possibilidade de outras pastas conforme necessidade
Essa estrutura facilita a manutenção, organização e escalabilidade do projeto.

O site deve conter os seguintes componentes essenciais:
-Logo da Uniube com link abrindo em uma nova aba
-Menu de navegação com links para todas as páginas do site
-Rodapé contendo o nome do aluno e RA
-Galeria de imagens relacionada ao tema escolhido
-Uso de listas e tabelas para organização de conteúdo
-Formulário de contato (ex: “Fale Conosco”)
Esses elementos garantem uma boa usabilidade e estrutura básica de um site profissional.

O site deve ser publicado utilizando:
GitHub Pages, Repositório hospedado no GitHub
Além disso:
O link do site deve ser enviado no diário de bordo
O repositório deve ser compartilhado com o professor (usuário indicado)

Criar um arquivo sitemap.xml
Enviar o sitemap para o Google Search Console
Isso ajuda o site a ser encontrado nos mecanismos de busca.

Criar o arquivo robots.txt
Define quais páginas podem ou não ser indexadas pelos motores de busca.

Implementar Google Analytics
Permite analisar o tráfego e comportamento dos usuários no site.

Implementar Google Tag Manager
Facilita a gestão de scripts e rastreamentos sem alterar diretamente o código.

O trabalho exige não só a criação de um site visualmente funcional, mas também a aplicação de práticas reais do mercado, como hospedagem, SEO e análise de dados, tornando o projeto mais completo e profissional.
